The Problem

Too many files, too little context 🤯

When you point Claude at a folder with dozens or hundreds of files, it tries to read everything. It burns through its context window on files that aren't relevant to the current task, then runs out of room for the actual work. Output quality drops. Sessions stall.

claude-context-limit.log
> Loading workspace directory...
... Read 142 files
! Context window at 85% capacity
... Reading /archive/old_version_v1.html
... Reading /archive/draft_notes.txt
✖ Error: Context Limit Reached. Unable to process current task instructions.

The Solution

Three components, one seamless workflow 🛠️

The Context Kit makes Claude read the right files, in the right order, and ignore the rest. Here is the step-by-step walkthrough of what makes it tick.

Global Instructions 1

Rules pasted into Cowork global settings. They tell Claude to apply a tiered access model to every folder—read canonical docs first, load domain files only when needed, and ignore archives.

Target File:
global-instructions.md

Folder Manifests 2

A _MANIFEST.md file dropped into any working folder. It scopes the workspace, explaining what is the source of truth, domains mapping, and what should be skipped.

Target File:
manifest-template.md

Navigator Skill 3

A powerful Cowork skill that enforces the orientation protocol. It forces Claude to read the manifest, handle conflict resolution, and intelligently scope sub-agents context.

Target Folder:
tiered-context-navigator/

Architecture

The Tier Model 🏗️

Context Kit classifies every file into one of three tiers. Claude knows exactly where to look based on these clear instructions.

Tier 1

Canonical

What it is

Source-of-truth documents, manifests, master references.

When Claude reads it

Always, before any work.

Tier 2

Domain-Relevant

What it is

Working files in domain subfolders.

When Claude reads it

Only when the task touches that domain.

Tier 3

Archival

What it is

Old versions, drafts, superseded files.

When Claude reads it

Only when you explicitly ask.

Rollout & Personalize

Make it yours ✨

You don't need to roll out everything at once. Gradually integrate Context Kit into your workflow. Customize it with your own preferences to give Claude a deep understanding of your style.

  • 1

    Pave the global rules

    Paste the `global-instructions.md` into Cowork settings. Fill placeholder sections with your coding style, document formats, and terminology.

  • 2

    Manifest your busy folders

    Add `_MANIFEST.md` to your most complex folders. If it improves output, start adding it everywhere.

  • 3

    Level up with the Skill

    Install the Navigator Skill into Cowork capabilities for strict enforcement of the tier model during multi-step execution.

Frequently Asked Questions ❓

Will Claude update manifests automatically?

No. You retain complete editorial control. Claude relies on you to define what is canonical. Periodically check that your Tier 1 lists are still accurate; stale references will degrade output quality.

Do I need a manifest for every folder?

Absolutely not. Manifests are only for folders where you perform regular Cowork tasks and there are enough files (roughly 10+) that Claude needs guidance. Small folders do not need one.

How do I handle obsolete files?

Keep them in the folder, but classify them or move them into a directory flagged as Tier 3 in your Manifest. The Navigator Skill will actively skip those files unless specifically instructed to retrieve them.
